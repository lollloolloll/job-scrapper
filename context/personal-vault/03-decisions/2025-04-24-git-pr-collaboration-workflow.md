# Git/PR 협업을 개인 역량 증거로 남긴다

## 결정

Git 명령어 자체보다 **브랜치·PR·리뷰·병합·피드백 반영을 이해하고 협업 흐름을 운영한 경험**을 역량 증거로 정리한다.

## 맥락

2025-03~05 고신호 날짜에서 `GitHub Issues 이해하기`, `PR 병합 절차`, `PR 병합 확인 방법`, `Git pull 오류 해결`, `Prettier 적용 문의`, `PR 피드백 반영 및 병합`이 반복된다.

## 선택 이유

- 초급 개발자 포트폴리오에서 Git 사용은 흔하지만, 실제 협업 흐름을 설명할 수 있는지는 차별점이다.
- 오류 해결, 병합 확인, 리뷰 반영까지 이어진 흔적이 있어 단순 명령어 암기를 넘는다.
- 소상상점 프로젝트와 직접 연결되어 면접 사례로 쓰기 좋다.

## 예상 결과

면접 답변은 “Git을 썼다”가 아니라 “PR 기준으로 변경 범위를 작게 나누고, 리뷰 반영 후 병합 상태를 확인하며 충돌/빌드 문제를 추적했다”로 표현한다.
