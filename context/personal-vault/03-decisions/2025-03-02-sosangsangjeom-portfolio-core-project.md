# 소상상점을 포트폴리오 핵심 프로젝트로 다룬다

## 결정

소상상점/소상공인 협업 플랫폼 경험은 단순 학습 기록이 아니라 **이력서·포트폴리오·면접에서 가장 먼저 꺼내는 핵심 프로젝트**로 정리한다.

## 맥락

2025-03-02의 고신호 기록에는 `소상공인 협력 플랫폼 장점`, `창업 인터뷰 질문 아이디어`가 함께 등장한다. 이후 Git 협업, API 연동, 컴포넌트, 배포/SEO, 고객 확보 관련 질문이 반복되면서 프로젝트가 단순 구현을 넘어 서비스 운영/사업 이해로 확장되었다.

## 선택 이유

- 약 8개월에 걸친 장기 프로젝트로 설명 밀도가 높다.
- Next.js/React 구현뿐 아니라 협업, API, Storybook, 빌드/배포, SEO, 고객 확보 전략까지 연결된다.
- 사용자의 학습 흐름이 “오류 질문 → 기능 구현 → 협업/운영 → 서비스 관점”으로 성장한 증거가 있다.

## 대안

- 여러 작은 프로젝트를 나열한다.
- 기술 스택 중심으로만 소개한다.

## 예상 결과

면접에서는 `문제 → 원인 → 해결 → 결과` 구조로 이야기하고, 이력서에서는 기술 나열보다 프로젝트 책임 범위를 강조한다.
