# 청소년 시설 근로장학생 경험과 기관 운영 시스템 개발 스토리

## 한 줄 정리

사용자는 청소년 시설 **쌍청문**에서 근로장학생으로 근무하며, 기관 운영에서 실제로 필요한 **공간대여 예약 시스템**과 **키오스크/대여 관리 시스템**을 혼자 풀스택으로 개발하고 운영한 경험이 있다.

## 왜 중요한 경험인가

이 경험은 단순한 개인 프로젝트가 아니라, 실제 기관 안에서 발생한 운영 문제를 발견하고 웹 시스템으로 해결한 사례다.

핵심 가치는 다음과 같다.

- 실제 사용처가 있는 프로젝트였다.
- 사용자는 기관 내부 근무자로서 업무 흐름과 불편을 가까이에서 볼 수 있었다.
- 요구사항을 외부에서 받은 것이 아니라, 현장 운영 맥락을 이해하며 기능으로 옮겼다.
- 프론트엔드, 백엔드, DB, 관리자 기능, 운영 환경까지 혼자 다뤘다.
- 만든 시스템이 기관 내부에서 운영되었고, 이후 다른 기관 맞춤형 수정 요청으로 이어졌다.

## 프로젝트 맥락

### 공간대여 예약 시스템

쌍청문에서 공간 예약 업무를 효율화하기 위해 만든 시스템이다.

이력서/면접에서 강조할 수 있는 포인트:

- 청소년 시설 내부 공간 예약 업무를 웹 시스템으로 전환한 경험
- 사용자 예약 신청과 관리자 예약 확인/관리 흐름 구현
- 예약 정책, 시간 제한, 승인/상태 관리, 게스트 예약, soft delete 같은 실제 운영 조건 반영
- Next.js, Supabase, PostgreSQL 기반 풀스택 개발 경험
- 단독 개발 및 운영 경험

안전한 표현:

> 청소년 시설 쌍청문에서 근로장학생으로 근무하며, 기관 내 공간 예약 업무를 효율화하기 위해 Next.js와 Supabase 기반 공간대여 예약 시스템을 단독으로 개발하고 운영했습니다.

### 키오스크/대여 관리 시스템

쌍청문에서 물품 대여와 키오스크성 운영 요구사항을 바탕으로 만든 시스템이다.

이력서/면접에서 강조할 수 있는 포인트:

- 사용자 키오스크 화면과 관리자 대시보드 구현
- 물품, 사용자, 대여 기록, 대기열, 만료 처리, 동의서, 통계 기능 구현
- Next.js, TypeScript, Drizzle ORM, SQLite 계열 로컬 DB, Docker Compose 기반 실행 환경 경험
- 단독 풀스택 개발 및 기관 내부 운영 경험

안전한 표현:

> 청소년 시설 운영 과정에서 필요한 물품 대여 관리를 위해 Next.js 기반 키오스크/대여 관리 시스템을 단독으로 개발했습니다. 사용자 대여 화면, 관리자 물품·사용자·대여 기록 관리, 대여 만료 처리, 대기열, 동의서, 통계 기능을 구현하며 실제 운영 요구사항을 반영했습니다.

### dby-ramen-kiosk 확장 스토리

`dby-ramen-kiosk`는 기존 키오스크 시스템을 본 쌍청문 당시 부장님이 다른 기관 관장님으로 이동한 뒤, 새 기관 환경에 맞게 수정 요청을 하면서 진행 중인 커스터마이징 작업이다.

이력서/면접에서 강조할 수 있는 포인트:

- 기존 기관 내부 시스템이 다른 기관에서도 관심을 받을 만큼 실사용성이 있었다.
- 단순 재개발이 아니라 기존 구조를 바탕으로 기관별 운영 방식에 맞게 수정하는 경험이다.
- 운영 현장의 요구사항을 듣고 화면, 데이터, 관리 기능, 테마를 조정하는 경험으로 연결된다.

안전한 표현:

> 기존 기관에서 사용하던 키오스크 시스템을 본 관계자의 요청으로, 다른 기관 환경에 맞춘 `dby-ramen-kiosk` 커스터마이징 작업을 진행하고 있습니다. 기존 대여 관리 구조를 바탕으로 기관별 운영 방식, 화면 구성, 테마, 데이터 관리 요구사항을 반영하고 있습니다.

## 자기소개서에 녹일 핵심 서사

이 경험은 사용자의 개발자 정체성을 설명하기 좋다.

> 저는 청소년 시설에서 근로장학생으로 근무하며 단순 보조 업무에 머무르지 않고, 현장에서 반복되는 운영 문제를 웹 시스템으로 해결해본 경험이 있습니다. 공간 예약과 물품 대여처럼 담당자 확인과 수기 관리가 필요한 업무를 보며, 이를 시스템화하면 기관과 사용자 모두에게 편해질 수 있다고 생각했습니다. 그래서 Next.js 기반 공간대여 시스템과 키오스크형 대여 관리 시스템을 혼자 기획하고 풀스택으로 구현했습니다. 이 과정에서 개발은 단순히 기능을 만드는 일이 아니라, 실제 업무 흐름과 사용자의 불편을 이해해 개선하는 일이라는 점을 배웠습니다.

## 면접 답변 포인트

### Q. 혼자 만든 프로젝트 중 가장 실사용에 가까운 경험은 무엇인가요?

> 청소년 시설 쌍청문에서 근로장학생으로 일하며 만든 공간대여 시스템과 키오스크/대여 관리 시스템입니다. 기관 내부에서 실제로 필요한 예약, 대여, 관리자 관리 기능을 혼자 풀스택으로 구현했고, 운영 과정에서 필요한 정책과 화면을 계속 반영했습니다.

### Q. 왜 이 프로젝트가 단순 개인 프로젝트와 다르다고 생각하나요?

> 실제 사용처와 운영 맥락이 있었기 때문입니다. 기능을 마음대로 정한 것이 아니라, 기관에서 공간을 예약하고 물품을 대여하는 흐름, 관리자가 기록을 확인해야 하는 상황, 사용자에게 어떤 화면이 필요한지를 보고 만들었습니다. 그래서 기술 구현뿐 아니라 도메인 이해와 운영 편의성을 함께 고민했습니다.

### Q. dby-ramen-kiosk는 어떤 의미가 있나요?

> 이전 기관에서 사용하던 키오스크 시스템을 좋게 보신 관계자분이 다른 기관으로 이동한 뒤, 해당 기관에 맞춘 수정 작업을 요청하면서 진행 중인 확장 버전입니다. 제가 만든 내부 운영 시스템이 다른 환경에서도 재사용될 수 있다는 점에서 의미가 있고, 기관마다 다른 요구사항을 기존 구조에 맞춰 반영하는 경험을 하고 있습니다.

## 이력서 bullet 후보

- 청소년 시설 쌍청문 근로장학생으로 근무하며 기관 내부 공간 예약과 물품 대여 업무를 효율화하기 위한 웹 시스템을 단독으로 개발·운영했습니다.
- Next.js와 Supabase 기반 공간대여 예약 시스템을 구현해 사용자 예약 신청, 관리자 예약 관리, 예약 정책, 게스트 예약, soft delete 등 실제 운영 요구사항을 반영했습니다.
- Next.js, Drizzle ORM, SQLite 계열 로컬 DB 기반 키오스크/대여 관리 시스템에서 사용자 대여 화면, 관리자 대시보드, 대여 기록, 대기열, 만료 처리, 동의서 기능을 구현했습니다.
- 기존 기관에서 사용하던 키오스크 시스템을 바탕으로, 다른 기관 환경에 맞춘 `dby-ramen-kiosk` 커스터마이징 작업을 진행하며 재사용 가능한 운영 시스템으로 확장하고 있습니다.

## 주의해서 표현할 점

- 정확한 운영 기간, 사용자 수, 실제 사용 범위는 추가 확인 전까지 정량 수치로 쓰지 않는다.
- “기관 전체에 공식 도입”처럼 과하게 말하지 않고, “기관 내부 운영용”, “실제 운영 요구사항 반영”, “운영 경험” 정도로 표현한다.
- `dby-ramen-kiosk`는 완료된 성과가 아니라 “진행 중인 커스터마이징 작업”으로 표현한다.
- 기관명 공개가 부담스러운 지원서에서는 “청소년 시설” 또는 “지역 청소년 시설”로 익명화할 수 있다.
