# 진로 적합도 신호

## 한 줄 요약

현재 증거 기준으로 사용자는 **프론트엔드 중심 + 실제 운영 문제 해결 + Product Engineer 지향 + AI-native workflow** 조합이 가장 자연스럽다.

## 강하게 맞는 신호

### 1. 프론트엔드 개발자

적합 신호:

- Next.js/React/TypeScript 프로젝트 evidence가 많다.
- Storybook, API 연동, OAuth, SEO, middleware, Vercel/빌드 이슈 경험이 있다.
- 소상상점이 가장 이력서 친화적인 대표 프로젝트다.

주의:

- 단순 UI 구현자로 보이지 않도록 API/배포/협업/도메인 이해를 같이 설명해야 한다.

### 2. Product Engineer 지향

적합 신호:

- 쌍청문 프로젝트는 실제 기관 운영 문제를 시스템화한 경험이다.
- 웹에이전시/MVP 프로젝트는 고객 유입, 퍼널, 상담 흐름을 고민한 경험이다.
- 소상상점은 소상공인/상점/협업 도메인 이해와 연결된다.

주의:

- 지원서 직함으로 바로 `Product Engineer`를 내세우면 회사에 따라 모호할 수 있다.
- 기본 타이틀은 프론트엔드로 두고, 자기소개에서 지향성을 설명하는 편이 안전하다.

### 3. 프론트엔드 중심 풀스택

적합 신호:

- Supabase, Drizzle, SQLite, API route, migration, 관리자 기능 경험이 있다.
- 단독 개발 프로젝트에서 프론트/DB/운영정책을 함께 다뤘다.

주의:

- 백엔드 전문 포지션처럼 말하면 깊은 서버/인프라 질문 부담이 커진다.
- “필요한 만큼 풀스택 구현”으로 표현하는 편이 안전하다.

### 4. AI-native workflow

적합 신호:

- Claude Code, Codex, OpenCode, Hermes Agent 등 에이전트 도구 사용 경험.
- Obsidian/Graphify 기반 개인 memory workflow 구축.
- AI를 디버깅, 코드 이해, 문서화, 계획 수립에 활용하는 습관.

주의:

- 메인 직무를 “AI 전문가”로 두는 것은 위험하다.
- 개발 역량을 보조하는 작업 방식으로 설명한다.

## 현재 피하는 것이 좋은 포지션

- 백엔드 전문 Java/Spring 포지션.
- DevOps/인프라 중심 포지션.
- AI/ML 엔지니어 포지션.
- 고성능/대규모 트래픽 서버 포지션.
- 순수 퍼블리싱만 반복하는 포지션.

## 최종 적합도 판단

```text
1순위: Next.js/React 프론트엔드 개발자
2순위: Product Engineer 지향 프론트엔드 개발자
3순위: 프론트엔드 중심 풀스택 개발자
보조 강점: AI-native workflow, 웹에이전시/MVP 실전 감각
```

## 추천 자기소개 핵심 문장

> Next.js/React 기반 프론트엔드 개발을 중심으로, 실제 기관 운영 시스템을 단독 구현한 풀스택 경험과 고객 유입/MVP 흐름을 이해한 Product Engineer 지향성을 가진 개발자입니다. AI 도구를 코드 이해, 디버깅, 문서화, 작업 계획에 활용하며 빠르게 학습하고 문제를 구조화하는 방식으로 성장하고 있습니다.
