# 선택지: 프론트엔드 중심 풀스택

## 판단

보조 포지셔닝으로 좋다. 메인 타이틀을 “백엔드까지 깊은 풀스택”으로 두기보다는, **프론트엔드 중심으로 필요한 백엔드/DB/운영 로직까지 구현한 경험**으로 설명하는 편이 안전하다.

## 강한 근거

- Venue Booking System에서 Supabase, migration, 예약 정책, 관리자 기능을 다뤘다.
- Ramen/Kiosk 시스템에서 Drizzle, SQLite 계열 DB, 대여/반납/대기열/만료 처리, 관리자 대시보드 경험이 있다.
- 실제 기관 운영 시스템을 단독으로 구현했다.

## 리스크

- 백엔드 전문 포지션으로 지원하면 깊은 서버 설계, 대규모 트래픽, 인증/권한 보안, 인프라 질문이 강하게 들어올 수 있다.
- 현재 대표 evidence는 Next.js/React 프론트엔드 쪽이 더 풍부하다.

## 안전한 소개 문장

> 프론트엔드를 중심으로 개발하면서, Supabase/Drizzle/SQLite/API route를 활용해 실제 운영에 필요한 백엔드와 DB 구조까지 구현해본 경험이 있습니다.

## 준비할 면접 주제

- REST/API route 기본
- DB table/migration 기본
- Supabase row/table 설계 경험
- Drizzle ORM 사용 경험
- 인증/권한의 기본 개념
- 프론트 payload와 DB column mismatch 디버깅 경험
