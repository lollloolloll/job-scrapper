# 선택지: AI-native workflow

## 판단

메인 직무 타이틀이 아니라 **개발자 작업 방식의 차별점**으로 쓰는 것이 가장 안전하다.

## 강한 근거

- Claude Code, Codex, OpenCode, Hermes Agent 등 여러 에이전트형 도구를 직접 사용했다.
- AI를 코드 이해, 오류 로그 해석, 디버깅 방향 검토, 작업 계획 수립, 문서화에 활용한다.
- Obsidian/Graphify 기반 개인 memory workflow를 구축했다.
- raw AI log를 compact index, curated note, graph로 정리해 이력서/면접/회고에 재사용한다.

## 리스크

- “AI 전문가”처럼 말하면 검증 부담이 커진다.
- “AI가 코드를 대신 만들었다”는 인상을 주면 실력 의심을 받을 수 있다.
- 도구 이름만 나열하면 가벼워 보인다.

## 안전한 소개 문장

> AI 도구를 단순 코드 생성용이 아니라 오류 로그 해석, 코드 이해, 디버깅 방향 검토, 작업 계획 수립, 문서화에 활용하며, 최종 코드는 직접 검토하고 프로젝트 맥락에 맞게 수정하는 방식으로 사용합니다.
