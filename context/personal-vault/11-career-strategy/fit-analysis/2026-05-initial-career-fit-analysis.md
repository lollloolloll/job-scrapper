# 2026-05 초기 진로 적합도 분석

## 결론

현재 가장 안전한 진로 포지셔닝은 다음이다.

> **Next.js/React 기반 프론트엔드 역량을 중심으로, 실제 운영 문제를 풀스택으로 해결해본 경험과 AI-native workflow를 가진 주니어 웹 개발자**

더 짧게 표현하면:

> **서비스 흐름을 이해하고 빠르게 구현하는 AI-native 프론트엔드/Product Engineer 지향 개발자**

현재 이력에서 가장 강한 축은 프론트엔드다. 다만 단순 UI 개발자보다는, 실제 기관 운영 시스템을 혼자 만들고 고객 유입/퍼널/수주 프로젝트까지 경험한 흔적이 있으므로 **Product Engineer 지향성**을 함께 드러내는 것이 좋다.

## 현재 자산 요약

### 1. 소상상점

대표 가치:

- 약 8개월 이상 이어진 소상공인 협업/상점 플랫폼 프론트엔드 프로젝트.
- Next.js/React/TypeScript 기반 사용자 웹 구현.
- 상점/그룹상점/협업/채팅/지도/온보딩/인증 화면.
- Axios API client, OAuth callback, token store.
- Storybook/Chromatic 기반 UI 컴포넌트 문서화.
- next-sitemap, robots, middleware, 네이버 소유권 인증, build/deploy 이슈 대응.
- GitHub feature/fix/hotfix branch, PR template, merge/rollback 흐름.

진로상 의미:

> 프론트엔드 개발자 지원 시 가장 먼저 보여줄 대표 프로젝트다.

### 2. 쌍청문 공간대여/키오스크 시스템

대표 가치:

- 청소년 시설 근로장학생으로 일하며 실제 운영 문제를 발견.
- 공간 예약 시스템과 키오스크/대여 관리 시스템을 단독 풀스택으로 개발.
- 예약 정책, 게스트 예약, soft delete, 대여 만료, 대기열, 관리자 기록 등 실제 운영 조건 반영.
- dby-ramen-kiosk는 기존 시스템을 본 관계자의 요청으로 다른 기관 환경에 맞춘 커스터마이징 작업으로 확장.

진로상 의미:

> 단순 토이 프로젝트가 아니라 실제 사용처와 운영 맥락이 있는 프로젝트라는 점에서 강력한 차별점이다.

### 3. 웹에이전시/MVP 솔루션

대표 가치:

- 친구와 웹에이전시 또는 MVP 솔루션 부업/창업 준비.
- 공식 홈페이지, 랜딩, 문의/상담, FAQ, 포트폴리오 페이지 구현.
- 고객 유입, 전환 흐름, 모객, 퍼널 설계 관점 경험.
- 실제 수주 프로젝트 진행 중.

주의:

- `pox-user-web`은 confidential 프로젝트로 다룬다.
- 고객명, 프로젝트명, URL, 화면 캡처, 상세 도메인, 매출/성과 공개 금지.
- 이력서에서는 “비공개 고객사 웹 프로젝트”, “계약상 상세 공개 불가”처럼 익명화한다.

### 4. AI-native workflow

대표 가치:

- Claude Code, Codex, OpenCode, Hermes Agent 등 여러 AI agent 사용 경험.
- 코드 이해, 디버깅 방향 검토, 작업 계획, 문서화, 지식 정리에 AI 활용.
- Obsidian/Graphify 기반 개인 memory workflow 구축.

주의:

- 메인 타이틀은 “AI 전문가”가 아니라 개발자 포지션으로 둔다.
- AI는 작업 방식의 차별점으로 설명한다.

## 진로 선택지 비교

| 선택지 | 적합도 | 강점 | 리스크 | 권장 사용법 |
|---|---:|---|---|---|
| 프론트엔드 개발자 | 높음 | Next.js/React/Storybook/API/SEO/Git evidence 풍부 | 단순 UI 구현자로 보일 위험 | 메인 지원 직무 |
| Product Engineer 지향 | 높음 | 운영 문제, 고객 흐름, MVP 감각과 잘 맞음 | 타이틀이 모호할 수 있음 | 자기소개/프로젝트 설명에 녹임 |
| 프론트엔드 중심 풀스택 | 중간-높음 | Supabase/Drizzle/DB/API route/단독 구현 경험 | 백엔드 전문성 질문 부담 | 보조 표현으로 사용 |
| 웹에이전시/MVP | 보조로 높음 | 고객 유입/퍼널/수주 경험 | 취업 이력서에서 사업 의지가 강해 보일 수 있음 | 실전 감각 스토리로 사용 |
| AI-native workflow | 보조로 높음 | 구체 도구와 memory workflow 있음 | “AI가 대신 개발” 인상 위험 | 성장 방식/문제해결 방식으로 사용 |

## 추천 최종 포지셔닝

### 1순위

> Next.js/React 기반 프론트엔드 개발자. 사용자 웹 구현, API 연동, Storybook 컴포넌트 문서화, SEO/배포 이슈 대응을 경험했고, 실제 기관 운영 시스템을 단독으로 구현하며 서비스 흐름과 운영 문제를 제품으로 옮기는 경험을 쌓았습니다.

### 2순위 확장

> Product Engineer 지향 프론트엔드 개발자. 단순 화면 구현을 넘어, 기관 운영 문제, 소상공인 협업 서비스, 고객 유입용 랜딩/상담 흐름을 직접 다루며 사용자 문제를 기능과 제품 구조로 연결하는 개발자를 지향합니다.

### 3순위 보조

> AI-native workflow에 익숙한 개발자. Claude Code, Codex, OpenCode, Hermes Agent 등 AI 도구를 코드 이해, 디버깅, 문서화, 작업 계획 수립에 활용하며, Obsidian/Graphify 기반 개인 memory workflow로 개발 경험을 재사용 가능한 지식으로 정리하고 있습니다.

## 지원 전략

### 우선 지원 직무

1. 주니어 프론트엔드 개발자
2. Next.js/React 프론트엔드 개발자
3. 웹 서비스 프론트엔드 개발자
4. Product Engineer 성향을 선호하는 초기 스타트업
5. MVP/웹서비스를 빠르게 만드는 SI/에이전시/스타트업 포지션

### 신중히 접근할 직무

- 백엔드 전문 Java/Spring 포지션
- DevOps/인프라 중심 포지션
- AI/ML 엔지니어 포지션
- 고성능/대규모 트래픽 백엔드 포지션
- 순수 퍼블리싱만 요구하는 포지션

## 이번 주 액션

1. 이력서 메인 타이틀을 `프론트엔드 개발자`로 고정한다.
2. 프로젝트 3개를 먼저 정리한다.
   - 소상상점
   - Venue Booking System
   - Ramen/Kiosk 대여 관리 시스템
3. 각 프로젝트를 `문제 / 내가 한 일 / 기술 스택 / 배운 점 또는 결과`로 정리한다.
4. confidential 프로젝트 공개 범위를 별도 문서화한다.
5. 면접 대표 스토리 3개를 준비한다.
   - 소상상점: Storybook/API/SEO/GitHub 협업
   - 쌍청문: 실제 기관 운영 문제를 시스템화
   - AI-native: AI를 활용한 디버깅/문서화/memory workflow
